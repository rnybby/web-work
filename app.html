Part One: Solidify Terminology

  • What is HTTP?
    • HTTP stands for Hypertext Transfer Protocol, it is how browsers and servers communicate.
  • What is URL?
    • URL stands for Universal Resource Locator and it is the address of a source.
  • What is DNS?
    • DNS stands for Domain Name System and it contains the IP address and hostname of a server like a phonebook
  • What is query string?
    • Query string provides extra information, that might be used to change a page.
  • What are two HTTP verbs and how are they different?
    • We have GET and POST. GET - gets some data from the server (most pages, search forms) and POST - send some data to the server (pages that change data on server).
  • What is an HTTP request?
    • When you point to webpage on a server your browser first sends a request to the server with the exact URL you want.
  • What is an HTTP response?
    • After the browser sends a request to the server, the server responses back with the exact HTML text for that page.
  • What is an HTTP header? Give a couple examples of request and response headers you have seen.
    • HTTP Header a field of an HTTP request or response that passes additional context and metadata about the request or response. For example, a request message can use headers to indicate it's preferred media formats, while a response can use header to indicate the media format of the returned body.
  • What are the processes that happen when you type “http://somesite.com/some/page.html” into a browser?
    • First, the hostname will turn into the IP address. Then, it'll connect to the IP address. Finally, using the HTTP Protocol ask for /some/page.html.
